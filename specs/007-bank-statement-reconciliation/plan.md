## Summary

Build 2 complete, net-new Accountant/Admin pages on the approved Mohasebaty design system: `bank-statement-import.html` (استيراد كشف حساب بنكي — client/account/period + upload + mapping summary + preview table + actions) and `bank-reconciliation.html` (مطابقة البنك والمستندات — KPI cards + a **two-panel split workspace** of bank transactions vs suggested documents + a match-details card + reconcile/ignore/create/clarify actions). All UI reuses existing components — **zero new CSS** — with Arabic/RTL static demo data, no backend, and **no matching computed**. The split workspace reuses the approved `.workspace` (1fr 1fr) layout from `document-review.html`. The 2 sidebar coming-soon items are converted to live links across every page, `index.html` cards/counters are updated (28→30 screens, 9→7 coming-soon), and contextual links are added from `bank-accounts.html` and `data-import.html`.

**Lesson carried from Specs 004–006:** every class used MUST already exist in `assets/css/output.css`; copy markup from the named reference pages; **do NOT use `.btn-rose`** (purged from `output.css`). A rogue-class sweep gates sign-off.

## Phase Execution Notes

### Split Workspace (the one genuinely-new layout — but reused, not new)
`bank-reconciliation.html` uses `<div class="workspace">` with: column 1 = a `.card` holding the **bank-transactions** table (6 columns); column 2 = a `.left-col` stack holding the **suggested-documents** `.card` (6 columns) + the **match-details** `.card` (معاملة البنك المحددة / المستند المقترح / الفرق / ملاحظات) + the 4 action buttons. This mirrors `document-review.html` exactly (preview column + `.left-col`), so it is on-design and responsive (collapses to one column on mobile via the existing `.workspace` breakpoint). **No new CSS.**

### No New JS Required
- Upload (`initDropzone`), any modal/overlay (`initOverlays`), selector toggles (`initSingleSelect`/`initFilterSelects`) — all in `boot()`. Selecting a transaction row to highlight it / surface its match details is a visual state only (a pre-authored match-details card); no matching computed.

### Zero-New-CSS Rule (hard requirement)
Every class used MUST already exist in `assets/css/output.css`. Copy markup from the reference pages; **do NOT use `.btn-rose`**. Verified by `quickstart.md` check (j).

### Sidebar / index / Contextual Links
- Convert the 2 `is-soon` spans (`استيراد كشف الحساب` upload icon → `bank-statement-import.html`; `التسوية البنكية` check-circle icon → `bank-reconciliation.html`) across all 28 existing pages via one Python script (preserve icons, remove `.sl-soon`). New pages mark their own item `active` + `aria-current="page"`.
- `index.html`: convert the 2 `is-soon` landing cards (استيراد كشف الحساب ~line 419, التسوية البنكية ~426) to live `<a class="landing-card">`; chip `9 وحدة`→`7 وحدة`; footer `28 شاشة مكتملة + 9 وحدة قادمة` → `30 شاشة مكتملة + 7 وحدة قادمة`.
- Contextual links: `bank-accounts.html` → `bank-statement-import.html` ("استيراد كشف حساب"); `data-import.html` → `bank-statement-import.html` ("استيراد كشف بنكي"). (The import page's "متابعة للمطابقة" already links to reconciliation.)
