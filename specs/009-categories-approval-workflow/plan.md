# Implementation Plan: Document Categories & Approval Workflow (Spec 009)

**Branch**: `009-categories-approval-workflow` | **Date**: 2026-06-11 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/009-categories-approval-workflow/spec.md`

## Summary

Build 2 complete, net-new Admin/Accounting pages on the approved Mohasebaty design system: `document-categories.html` (تصنيفات المستندات — client/global-template, 4 category groups, an 8-column category→account/VAT/visibility table, add-category modal, usage card) and `approval-workflow.html` (مسار الاعتماد — 3 workflow switches, a 6-column approval-rules table, a 4-step workflow visual, add-rule modal, pending-approvals section). All UI reuses existing components — **zero new CSS** — Arabic/RTL, static data, no backend/enforcement; modals reuse the existing `.scrim` pattern. The 2 sidebar coming-soon items are converted to live links across every page, `index.html` cards/counters are updated (32→34 screens, 5→3 coming-soon), and contextual links are added from `settings.html` and `document-review.html`.

**Lesson carried from Specs 004–008:** every class used MUST already exist in `assets/css/output.css`; copy markup from the named reference pages; **do NOT use `.btn-rose`** (purged → `.btn-amber`/`.btn-outline`). A rogue-class sweep gates sign-off. (Also: sidebar wiring uses the **label-based line fixer** proven in the Spec 008 repair — never a spanning/DOTALL regex.)

### Zero-New-CSS Rule (hard requirement)
Every class used MUST already exist in `assets/css/output.css`. Copy markup from the reference pages; **do NOT use `.btn-rose`**. Verified by `quickstart.md` check (j).

- Convert the 2 `is-soon` spans (`تصنيفات المستندات` → `document-categories.html`; `سير الاعتماد` → `approval-workflow.html`) across all 32 existing pages using the **label-based line fixer** from the Spec 008 repair (locate the label line, rewrite its immediate wrapper — NEVER a spanning/DOTALL regex). New pages mark their own item `active` + `aria-current="page"`.
- `index.html`: convert the 2 `is-soon` landing cards (تصنيفات المستندات ~line 290, سير الاعتماد ~line 437) to live `<a class="landing-card">`; chip `5 وحدة`→`3 وحدة`; footer `32 شاشة مكتملة + 5 وحدة قادمة` → `34 شاشة مكتملة + 3 وحدة قادمة`.
