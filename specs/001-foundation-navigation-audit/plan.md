# Implementation Plan: Foundation / Navigation / Audit

**Branch**: `001-foundation-navigation-audit` | **Date**: 2026-06-07 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `specs/001-foundation-navigation-audit/spec.md`

## Summary

Audit the 16 existing entry points (`index.html` + 15 pages), confirm they open correctly in Arabic
RTL with working local assets and no broken internal links, then prepare the internal (accountant/admin)
sidebar for the next phase by adding seven new Arabic navigation groups containing 22 upcoming modules.
Upcoming modules are rendered as non-navigating "قريباً" (coming soon) items so the navigation is
"ready" without 404s and without creating empty pages. `index.html` is updated to reflect the full
product scope (15 current + 22 upcoming). The approved design system is reused as-is; at most one small
CSS utility (a dimmed "coming soon" sidebar item + pill) is added to the Tailwind input layer and the
stylesheet is rebuilt. No JavaScript and no page content generation is introduced.

## Technical Context

**Language/Version**: HTML5, CSS3, vanilla JavaScript (ES5-style IIFE, matching existing `main.js`)
**Primary Dependencies**: Tailwind CSS 3.4 built locally (`npm run build:css`); local IBM Plex Sans
Arabic / Mono fonts. No CDN, no Bootstrap, no React/Vue/Angular/Next.
**Storage**: N/A — static prototype; all data hardcoded in HTML.
**Testing**: Manual verification in a browser / Live Server (no automated test framework; consistent with
the constitution's frontend-only, tests-optional stance). A repeatable manual audit checklist is the gate.
**Target Platform**: Modern evergreen browsers; desktop-first, responsive to tablet and mobile.
**Project Type**: Static multi-page web frontend (MPA). No SPA, no backend, no JS-rendered content.
**Performance Goals**: Instant static loads; zero runtime content generation; no new network dependencies.
**Constraints**: Arabic-only, RTL-first (`lang="ar" dir="rtl"`); do not redesign the approved style; do
not break/remove existing pages, sections, links, or asset paths; do not create empty/skeleton pages;
upcoming links must not 404.
**Scale/Scope**: 16 existing entry points to audit; sidebar present and duplicated on 15 pages; 7 new
nav groups; 22 upcoming module placeholders; `index.html` landing scope updated to 37 modules.
