# منصة محاسبتي — Frontend

Arabic RTL outsourced accounting / bookkeeping platform — static HTML / CSS /
Tailwind frontend, ready to plug into Django templates.

## Pages

| الصفحة | المسار |
|---|---|
| الصفحة الرئيسية (فهرس النماذج) | `/index.html` |
| لوحة العميل | `/pages/client-dashboard.html` |
| رفع مستند جديد | `/pages/upload-document.html` |
| صندوق المستندات (للمحاسب) | `/pages/accountant-inbox.html` |
| مراجعة وتصنيف المستند | `/pages/document-review.html` |
| لوحة التقارير المالية (للإدارة) | `/pages/admin-financial-dashboard.html` |

## Tech

- HTML + CSS + Tailwind CSS v3 (locally built — **no CDN**)
- Vanilla JS only for small UI interactions (no framework, no content generation)
- IBM Plex Sans Arabic + IBM Plex Mono — self-hosted from `@fontsource` npm packages

## Setup

```bash
npm install
npm run build      # copies fonts + builds output.css
```

`npm run build` runs both `setup:fonts` (copies the seven woff2 font files from
`node_modules/@fontsource/*` into `assets/fonts/`) and `build:css` (compiles the
final CSS into `assets/css/output.css`).

## Development

```bash
npm run watch:css   # rebuild CSS on change
npm run serve       # python3 -m http.server 8000 (then open http://localhost:8000)
```

You can also just open `index.html` directly in the browser — the project is
fully static and needs no server to view.

## Django integration

Each HTML file marks its repeated regions with comments:

```html
<!-- BLOCK: sidebar -->
...
<!-- /BLOCK -->

<!-- BLOCK: header -->
...
<!-- /BLOCK -->
```

When porting to Django:
1. Move `<!-- BLOCK: sidebar -->` content into `templates/partials/sidebar.html`
   and replace with `{% include 'partials/sidebar.html' %}`.
2. Same for the header block (it varies per page — make it a base template with
   blocks like `{% block page_title %}{% endblock %}`).
3. Move `assets/` under your static files dir and reference with `{% static %}`.
4. Replace hardcoded sample data (KPIs, rows, etc.) with template variables.

## JavaScript

`assets/js/main.js` has these feature-detected modules (each guarded by a
querySelector check so the file is safe on every page):

| Module | Where used |
|---|---|
| `initSidebarMobile` | All pages — mobile burger toggles `.sidebar` |
| `initBellDropdown` | Header `.notif-panel` on every page |
| `initDropzone` | Upload page — drag/drop visuals + file preview |
| `initSingleSelect` | Type cards, segmented controls, toggle radios, tabs |
| `initFilterSelects` | Inbox filters |
| `initOverlays` | `[data-open]` triggers for modals/drawers |
| `initBulkBar` | Inbox row checkboxes ↔ bulk-actions bar |

No content is ever generated by JS — every visible string is in the HTML.

## Assumptions / Notes

- The original prototype design (under `/tmp/design-extract/`) uses only a top
  header. The user asked for a left sidebar in addition — implemented as a
  separate `.sidebar` next to the existing header on every page.
- Fonts are self-hosted (~250KB total for the seven weights). License is OFL.
- All chart SVGs (bar chart, donut, sparklines) are **precomputed and inlined**
  as static `<rect>`, `<circle>`, `<path>` elements. No JS chart libraries.
